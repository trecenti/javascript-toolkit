Dependency Loading

Tools that enable your JavaScript files to import other modules, either dynamically or on build-time.

Browserify

Browserify lets you load modules by bundling the dependencies in the required order, producing a customized JavaScript asset. It can require CommonJs-compliant modules, and with the help of browserify-shim non-Commonjs modules (like AMD and global scripts).

Links

  • http://browserify.org/

Closure Compiler

Closure Compiler bundles and minify your JavaScript with type annotations, allowing bad code elimination and other optimizations. It also let you create separate bundled files to load asynchronously when your app needs, instead of just loading a big single file on the client.

Links

  • https://developers.google.com/closure/compiler/

RequireJS

RequireJS is a AMD-compliant module loader, allowing you to declare dependencies between modules and providing them on the correct dependency order.

Links

  • http://requirejs.org/

SystemJS

AMD, CJS & ES6 spec-compliant module loader.

Links

  • https://github.com/systemjs/systemjs

script.js

Async JavaScript loader & dependency manager

Links

  • https://github.com/ded/script.js

Webpack

Webpack is a module bundler tool, that generates your assets with the modules required in order. It can make use of AMD-compliant, CommonJs-compliant modules, ES6 modules, script-tags and custom-defined bundles. There are costumization options for many of the aspects of the generated assets, as well as pre-processing files before bundling in.

Links

  • https://webpack.github.io/
